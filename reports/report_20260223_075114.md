### CRIT-001: PHPInfo Page Publicly Accessible

**Affected Component:** `/phpinfo.php`  
**CVSS v3.1 Score:** 8.6 (High) - AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:N/A:N  
**Discovery Tools:** gobuster, ffuf

#### Technical Description

The PHP information disclosure page (`phpinfo.php`) is publicly accessible without authentication. This diagnostic page exposes comprehensive server configuration details including:

- PHP version and compilation options
- Loaded extensions and their versions
- Server paths and directory structure
- Environment variables
- Database connection details (if configured)
- Security settings (disabled functions, open_basedir, etc.)
- Session configuration
- File upload settings

#### Evidence

```
Tool: gobuster
Finding: phpinfo.php (Status: 200) [Size: 80042]

Tool: ffuf  
Finding: phpinfo.php accessible (HTTP 200)
Response Size: 80,042 bytes
```

The large response size (80KB) confirms a full phpinfo() output rather than a stub file.

#### Impact Analysis

**Immediate Risks:**
1. **Version Disclosure:** Attackers can identify exact PHP version and target known CVEs
2. **Path Disclosure:** Full filesystem paths enable path traversal attack construction
3. **Configuration Intelligence:** Security settings reveal attack surface (e.g., disabled functions, allowed protocols)
4. **Extension Enumeration:** Loaded modules indicate available attack vectors (e.g., curl, file_get_contents capabilities)
5. **Credential Exposure:** Database credentials may be visible in environment variables

**Attack Scenarios:**
- Attacker identifies PHP 7.x with known unserialize() vulnerability
- Server paths used to construct LFI/RFI payloads
- Disabled functions list used to identify available code execution methods
- Session configuration exploited for session fixation attacks

#### Proof of Concept

```bash
curl -s https://dvwa.csalab.app/phpinfo.php | grep "PHP Version"
# Expected: Full PHP version disclosure

curl -s https://dvwa.csalab.app/phpinfo.php | grep "_SERVER\[\"DOCUMENT_ROOT\"\]"
# Expected: Full server path disclosure
```

#### Remediation Steps

**Immediate Actions (Priority 1 - Within 24 hours):**
1. Delete `/phpinfo.php` from the web root:
   ```bash
   rm /var/www/html/phpinfo.php
   ```

2. Verify removal:
   ```bash
   curl -I https://dvwa.csalab.app/phpinfo.php
   # Expected: HTTP 404
   ```

**Short-term Actions (Priority 2 - Within 1 week):**
3. Implement web server-level blocking for common diagnostic files:
   ```apache
   # Apache .htaccess
   <FilesMatch "^(phpinfo|info|test)\.php$">
       Require all denied
   </FilesMatch>
   ```

   ```nginx
   # Nginx configuration
   location ~* ^/(phpinfo|info|test)\.php$ {
       deny all;
       return 404;
   }
   ```

4. Scan for similar diagnostic files:
   ```bash
   find /var/www/html -name "*info*.php" -o -name "test*.php"
   ```

**Long-term Actions (Priority 3 - Within 1 month):**
5. Implement file integrity monitoring to detect unauthorized diagnostic files
6. Add automated security scanning to CI/CD pipeline to prevent deployment of diagnostic files
7. Create secure administrative interface for authorized diagnostics with:
   - IP whitelist restrictions
   - Strong authentication
   - Audit logging

**Validation:**
```bash

nmap -p 443 --script http-enum dvwa.csalab.app | grep phpinfo
# Expected: No results

# Test from external IP
curl -I https://dvwa.csalab.app/phpinfo.php
# Expected: HTTP 404 or 403
```

### HIGH-002: .htpasswd File Publicly Accessible

**Affected Component:** `/.htpasswd`  
**CVSS v3.1 Score:** 7.5 (High) - AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:N/A:N  
**Discovery Tools:** ffuf

#### Technical Description

The Apache password file (`.htpasswd`) is accessible via HTTP. This file contains usernames and hashed passwords used for HTTP Basic Authentication. While passwords are hashed (typically using bcrypt, MD5, or SHA), exposure enables offline brute-force attacks.

#### Evidence

```
Tool: ffuf
.htpasswd file discovered (HTTP 200)
```

#### Impact Analysis

**Credential Compromise Risk:**

Typical `.htpasswd` format:
```
admin:$apr1$abc123$hashedpassword
user1:$apr1$def456$hashedpassword
testuser:$apr1$ghi789$hashedpassword
```

**Attack Process:**
1. Download `.htpasswd` file
2. Extract usernames (plaintext)
3. Identify hash algorithm (apr1 = Apache MD5)
4. Offline brute-force attack using hashcat/john

**Hashcat Example:**
```bash
# Download file
curl -s https://dvwa.csalab.app/.htpasswd > htpasswd.txt

# Crack with hashcat
hashcat -m 1600 htpasswd.txt /usr/share/wordlists/rockyou.txt

# Expected time to crack:
# Weak passwords (password123): Minutes
# Medium passwords (P@ssw0rd): Hours to days
# Strong passwords (random 16+ chars): Infeasible
```

**Business Impact:**
- Unauthorized access to protected areas
- Credential reuse on other systems
- User enumeration
- Compliance violations

#### Proof of Concept

```bash
# Download .htpasswd
curl -s https://dvwa.csalab.app/.htpasswd

# Expected output (if vulnerable):
admin:$apr1$abc123$...
user:$apr1$def456$...

# Extract usernames
curl -s https://dvwa.csalab.app/.htpasswd | cut -d: -f1

# Attempt offline cracking
hashcat -m 1600 -a 0 htpasswd.txt rockyou.txt --show
```

#### Remediation Steps

**Immediate Actions (Priority 1 - Within 1 hour):**

1. **Block Access to .ht* Files:**
   ```apache
   # Apache - Add to main config or .htaccess
   <FilesMatch "^\.ht">
       Require all denied
   </FilesMatch>
   ```

   ```nginx
   # Nginx
   location ~ /\.ht {
       deny all;
       return 404;
   }
   ```

2. **Verify Protection:**
   ```bash
   curl -I https://dvwa.csalab.app/.htpasswd
   # Expected: 403 or 404
   
   curl -I https://dvwa.csalab.app/.htaccess
   # Expected: 403 or 404
   ```

**Short-term Actions (Priority 2 - Within 4 hours):**

3. **Move .htpasswd Outside Web Root:**
   ```bash
   # Current (vulnerable)
   /var/www/html/.htpasswd
   
   # Secure location
   /etc/apache2/.htpasswd
   
   # Move file
   mv /var/www/html/.htpasswd /etc/apache2/.htpasswd
   chmod 640 /etc/apache2/.htpasswd
   chown root:www-data /etc/apache2/.htpasswd
   ```

4. **Update Apache Configuration:**
   ```apache
   <Directory /var/www/html>
       AuthType Basic
       AuthName "Restricted Area"
       AuthUserFile /etc/apache2/.htpasswd  # Updated path
       Require valid-user
   </Directory>
   ```

5. **Restart Apache:**
   ```bash
   sudo systemctl restart apache2
   # or
   sudo service apache2 restart
   ```

**Short-term Actions (Priority 3 - Within 24 hours):**

6. **Rotate All Passwords:**
   ```bash
   # Assume compromise - change all passwords
   htpasswd -B /etc/apache2/.htpasswd admin
   # Enter new strong password
   
   # For all users
   for user in $(cut -d: -f1 /etc/apache2/.htpasswd); do
     echo "Resetting password for: $user"
     htpasswd -B /etc/apache2/.htpasswd $user
   done
   ```

7. **Implement Strong Password Policy:**
   - Minimum 16 characters
   - Use bcrypt hashing (-B flag)
   - No dictionary words
   - Regular rotation (90 days)

8. **Monitor for Exploitation:**
   ```bash
   # Check Apache access logs for .htpasswd requests
   grep "\.htpasswd" /var/log/apache2/access.log
   
   # Check for authentication failures
   ```
